# ブロッカー報告テンプレート

## 🚨 緊急度別報告フォーマット

### 🔴 P0 - Critical（即座対応）

```markdown
🚨 P0 CRITICAL BLOCKER 🚨

## 基本情報
- **報告者**: [エージェント名]
- **報告時刻**: YYYY/MM/DD HH:MM:SS
- **ブロッカーID**: BLOCK-[YYYYMMDD]-[NNN]
- **関連タスク**: [タスクID・名称]

## 緊急事態詳細
**問題概要**: 
[1行で問題を要約]

**現在の状況**:
- システム状態: [停止/部分機能/性能劣化]
- 影響範囲: [ユーザー数・システム・ビジネス]
- 発見経緯: [どのように問題を発見したか]
- 現在時刻での被害: [具体的影響・損失]

**即座の危険**:
- 放置時のリスク: [最悪シナリオ]
- 拡大可能性: [他への波及リスク]
- 時間的制約: [いつまでに解決必要か]

## 初期対応済み事項
**応急処置**:
- [ ] [実施した緊急対応1]
- [ ] [実施した緊急対応2]
- [ ] [実施した緊急対応3]

**情報収集**:
- ログ確認: [確認済みログ・エラー内容]
- 影響調査: [被害範囲・ユーザー影響]
- 再現性: [問題再現の可否・条件]

## 必要な緊急支援
**技術支援**:
- 必要な専門知識: [分野・技術]
- 必要な権限: [システム・承認]
- 必要なツール: [緊急調達必要項目]

**判断依頼**:
- 上位決定事項: [経営・戦略判断]
- 外部連絡: [顧客・パートナー通知]
- リソース配分: [人員・予算緊急投入]

**連絡事項**:
- 通知済み: [連絡完了者・方法]
- 通知必要: [追加連絡必要者]
- エスカレーション: [上位報告必要性]

## 解決計画
**短期対応** (1時間以内):
1. [緊急対応ステップ1] (担当: [名前])
2. [緊急対応ステップ2] (担当: [名前])
3. [影響最小化措置] (担当: [名前])

**中期対応** (24時間以内):
1. [根本原因分析] (担当: [名前])  
2. [恒久対策実施] (担当: [名前])
3. [再発防止策] (担当: [名前])

**フォローアップ**:
- 進捗報告: [XX分毎に状況共有]
- 完了確認: [解決判定基準]
- 事後検証: [ポストモーテム計画]
```

### 🟡 P1 - High（1時間以内対応）

```markdown
⚠️ P1 HIGH PRIORITY BLOCKER ⚠️

## 基本情報
- **報告者**: [エージェント名]
- **発生時刻**: YYYY/MM/DD HH:MM
- **関連プロジェクト**: [プロジェクト名]
- **影響タスク**: [影響を受けるタスクリスト]

## 問題詳細
**ブロッカー内容**:
[具体的な問題・障害の詳細説明]

**根本原因（推定）**:
- 技術的要因: [システム・コード・設定]
- プロセス要因: [手順・承認・連携]
- 外部要因: [API・インフラ・依存関係]
- 人的要因: [スキル・情報・コミュニケーション]

**影響分析**:
- 作業影響: [停止・遅延するタスク]
- 期限影響: [プロジェクト期限への影響度]
- 品質影響: [成果物品質への影響]
- チーム影響: [他メンバーへの影響]

## 対処実績
**試行した解決策**:
1. **[解決策1]**
   - 実施内容: [具体的行動]
   - 結果: [効果・成果]
   - 学習: [得られた知見]

2. **[解決策2]**  
   - 実施内容: [具体的行動]
   - 結果: [効果・課題]
   - 次策: [改良・代替案]

**調査・分析実績**:
- 技術調査: [実施した技術的調査]
- データ収集: [ログ・メトリクス・エビデンス]
- 類似事例: [過去事例・解決方法参照]
- 外部情報: [文档・コミュニティ・サポート]

## 解決に必要な支援
**専門知識・スキル**:
- 必要分野: [技術領域・専門性]
- 支援者候補: [エキスパート名]
- 学習時間: [習得に必要な時間]

**権限・承認事項**:
- システム権限: [必要なアクセス権]
- 予算承認: [ツール・サービス購入]
- 方針決定: [技術選択・アプローチ変更]

**リソース・ツール**:
- 追加人員: [作業支援・並行対応]
- 開発環境: [テスト・検証環境]
- 外部サービス: [API・ツール・コンサル]

**時間・工数**:
- 解決見積もり: [必要時間・工数]
- 優先度調整: [他タスクとの兼ね合い]
- 期限調整: [プロジェクトスケジュール]

## 代替案・回避策
**短期回避策**:
- 暫定対応: [一時的解決方法]
- 機能制限: [縮小範囲での実現]
- 手動対応: [自動化未実装での運用]

**代替アプローチ**:
- 技術変更: [異なる技術・手法選択]
- 仕様変更: [要求・設計の調整]
- スケジュール: [フェーズ・順序の変更]

**影響軽減策**:
- 並行作業: [影響を受けない作業の継続]
- 事前準備: [解決後の迅速復旧準備]
- コミュニケーション: [関係者への状況共有]

## 学習・改善機会
**技術学習**:
- 新技術習得: [必要な技術・スキル]
- 深い理解: [既存技術の理解深化]
- ベストプラクティス: [業界標準・推奨手法]

**プロセス改善**:
- 手順見直し: [作業プロセスの改善]
- チェック強化: [事前確認・検証手法]
- 自動化推進: [手動作業の自動化]

**予防策**:
- 監視強化: [早期発見の仕組み]
- テスト強化: [事前検証の充実]
- ドキュメント: [知識・手順の記録]
```

### 🟢 P2 - Medium（4時間以内対応）

```markdown
🔧 P2 MEDIUM PRIORITY BLOCKER

## 基本情報
- **報告者**: [エージェント名]
- **発生時期**: [いつ頃から問題を感じているか]
- **頻度**: [daily/weekly/occasional/rare]
- **緊急性**: [なぜP2と判定したか]

## 問題詳細
**具体的な問題**:
[何に困っているか・どういう時に発生するか]

**業務への影響**:
- 効率への影響: [作業速度・品質への影響度]
- ストレス影響: [精神的負担・フラストレーション]
- 学習への影響: [成長・スキル向上への阻害]

**発生パターン**:
- 発生条件: [どのような時・環境で起きるか]
- 再現性: [必ず起きる/時々/条件次第]
- 回避可能性: [避ける方法があるか]

## 現在の対処法
**暫定対応**:
[現在どのようにして問題を回避・対処しているか]

**所要コスト**:
- 時間コスト: [対処に要する時間]
- 品質コスト: [品質低下・妥協点]
- 機会コスト: [本来できたはずのこと]

**限界・課題**:
- 対処法の問題: [現状の対処法の限界]
- 持続可能性: [いつまで続けられるか]
- 拡張性: [規模拡大時の問題]

## 理想的な解決策
**希望する改善**:
[どのようになれば問題が解決するか]

**参考事例**:
- 他チーム: [他で見た良い事例]
- 業界事例: [ベストプラクティス]
- ツール: [解決に役立ちそうなツール]

**学習・成長機会**:
[この問題解決を通じて得られる学び・スキル]

## 期待効果
**効率改善**:
- 時間短縮: [解決による時間削減効果]
- 品質向上: [成果物品質の向上]
- ストレス軽減: [精神的負担の軽減]

**チーム貢献**:
- 知識共有: [他メンバーへの学習貢献]
- プロセス改善: [チーム全体への改善効果]
- イノベーション: [新しいアプローチの可能性]

**長期価値**:
- スキル習得: [個人の成長・能力向上]
- 再利用性: [他の問題への応用可能性]
- 組織資産: [組織のナレッジ・仕組み化]

## 解決アプローチ
**調査計画**:
1. [情報収集・リサーチ] (期間: X日)
2. [技術検証・実験] (期間: X日)  
3. [解決策評価・選択] (期間: X日)

**実装計画**:
1. [設計・準備] (工数: Xh)
2. [実装・構築] (工数: Xh)
3. [テスト・検証] (工数: Xh)
4. [デプロイ・展開] (工数: Xh)

**リスク・制約**:
- 技術的リスク: [想定される技術的課題]
- リソース制約: [人・時間・予算制約]
- 優先度競合: [他タスクとのバランス]

## 支援要請
**情報・知識**:
- 技術情報: [必要な技術資料・情報]
- 経験談: [類似問題の解決経験]
- 外部情報: [書籍・記事・コミュニティ]

**人的支援**:
- レビュー: [設計・実装のレビュー]
- ペアワーク: [一緒に問題解決]
- 相談: [定期的な進捗相談]

**環境・ツール**:
- 開発環境: [検証・テスト環境]
- ツール: [必要なソフトウェア・サービス]
- 権限: [システム・データアクセス]
```

## ブロッカー進捗追跡

**ブロッカーID**: BLOCK-[YYYYMMDD]-[NNN]
**最終更新**: YYYY/MM/DD HH:MM

### 📊 基本情報
- ステータス: [新規/対応中/解決待ち/完了]
- 緊急度: P[0-3]
- 報告者: [名前]
- 担当者: [名前]
- 作成日: YYYY/MM/DD
- 期限: YYYY/MM/DD

### 📈 進捗記録
**[日時]** - [担当者]
- 実施内容: [具体的作業・調査内容]
- 成果: [得られた結果・発見]
- 課題: [新たに判明した問題]
- 次回: [次のアクション・予定]

### 🎯 完了判定基準
- [ ] [完了条件1]
- [ ] [完了条件2] 
- [ ] [完了条件3]
- [ ] [再発防止策実装]
- [ ] [ドキュメント更新]

### 📚 学習記録
- **技術学習**: [習得した技術・知識]
- **プロセス改善**: [改善されたプロセス・手順]
- **予防策**: [同様問題の予防方法]

```

- **効果**: [解決による改善効果]
- **所要時間**: [発見から解決までの時間]

### 🔍 根本原因
- **直接原因**: [直接的な原因]
- **根本原因**: [5 Whys分析結果]
- **システム要因**: [システム・プロセス的要因]
- **学習ポイント**: [今後の教訓]

### 🛡️ 再発防止策
1. **監視強化**: [早期発見の仕組み]
2. **プロセス改善**: [手順・チェック改善]
3. **自動化**: [手動作業の自動化]
4. **教育・共有**: [知識・経験の共有]

### 📊 効果測定
- **解決時間**: [目標 vs 実績]
- **品質影響**: [なし/軽微/あり]
- **コスト影響**: [追加コスト・時間]
- **満足度**: [報告者・チーム満足度]

### 🎓 学習・成長
- **新スキル**: [習得した技術・知識]
- **改善提案**: [プロセス・ツール改善案]
- **横展開**: [他への応用可能性]
- **感謝**: [協力・支援への感謝]
```
