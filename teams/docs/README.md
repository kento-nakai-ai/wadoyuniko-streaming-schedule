# Docs（共通ドキュメント）チーム

## チーム概要と責任範囲

**バイブコーディングローンチプロジェクト**の情報基盤を担当し、全チーム共通で参照する重要ドキュメントの管理・運用を行います。プロジェクト成功に必要な情報の一元管理と効率的な情報共有を実現します。

### 主要メンバーと役割
- **ドキュメントマネージャー**: 全体情報戦略・ドキュメント品質管理
- **テクニカルライター**: 技術仕様・API・システムアーキテクチャ文書化
- **プロジェクトアーキビスト**: プロジェクト履歴・意思決定記録・ナレッジ蓄積

### 情報管理責任領域
- **製品仕様管理**: バイブコーディング・プロンプトエンジニアリング仕様の一元管理
- **技術仕様統括**: システムアーキテクチャ・API・開発環境の文書化・更新
- **プロジェクト戦略文書**: ローンチ戦略・マイルストーン・KPI管理文書の維持
- **緊急時対応**: 危機管理プロトコル・連絡体制・対応手順の文書化
- **品質保証**: 全ドキュメントの正確性・最新性・アクセシビリティ確保
- **知識共有**: チーム間情報共有・ベストプラクティス・レッスンズラーンド蓄積

## 緊急時対応プロトコルと危機管理

### レベル1: 軽微な問題（チーム内解決）
- **対象**: 個別タスク遅延・軽微な技術問題・日常的調整
- **対応**: 担当者自己解決・チーム内相談・即座改善
- **報告**: Slack簡易報告・日次会議で共有

### レベル2: 中程度の問題（チーム間調整必要）
- **対象**: スケジュール調整・品質問題・チーム間連携課題
- **対応**: チームリーダー間調整・PM報告・戦略修正
- **報告**: 正式報告書・改善計画・フォローアップ

### レベル3: 重大な問題（プロジェクト全体影響）
- **対象**: 売上影響・技術的重大問題・外部要因・緊急事態
- **対応**: 緊急会議招集・全チーム総動員・危機管理体制
- **報告**: 即座PM連絡・緊急対応チーム結成・ステークホルダー報告

### 危機管理連絡体制
1. **第一報**: Slack緊急チャンネル（即座通知）
2. **詳細報告**: 専用フォーム・影響分析・対策提案
3. **継続フォロー**: 定期進捗報告・改善効果測定・再発防止
