# わどユニコプロジェクト - チーム別ドキュメント管理システム

## プロジェクト概要

**わどユニコプロジェクト**は、2025年8月28-30日にローンチする**バイブコーディング（Vibe Coding）**AI開発教育コースのビジネスローンチプロジェクトです。売上目標3000万円、受講者100名を目指す大規模なマーケティング・教育事業です。

### 基本情報
- **プロジェクト名**: わどユニコプロジェクト（バイブコーディングローンチ）
- **期間**: 2025年8月14日 - 9月7日
- **売上目標**: 3000万円
- **受講者目標**: 100名
- **主催**: わど × ユニコーン社

## プロジェクトスケジュール

### Phase 1: 認知拡大（8/14 - 8/20）
**8月14日: 大規模コラボレーション発表**
- 🎯 目標: プロジェクト認知度向上
- 📢 チャネル: X、YouTube、Instagram
- 📊 KPI: フォロワー増加、エンゲージメント向上

### Phase 2: リード獲得（8/21 - 8/27）
**8月21日: ローンチポストキャンペーン開始**
- 🎯 目標: セミナー登録者1000名獲得
- 📢 チャネル: 全SNS、メール、LINE
- 📊 KPI: セミナー登録数、リード獲得数

### Phase 3: コンバージョン（8/28 - 8/30）
**8月28-30日: 3日間セミナー実施**
- 🎯 目標: 100名の受講者獲得、3000万円売上
- 📢 チャネル: ライブ配信、個別相談
- 📊 KPI: 参加率、成約率、売上

### Phase 4: フォローアップ（9/1 - 9/7）
**9月1-7日: アフターフォロー**
- 🎯 目標: 顧客満足度向上、追加売上
- 📢 チャネル: Discord、メール、個別サポート
- 📊 KPI: 顧客満足度、リピート率

## ドキュメント利用ガイド

### 📖 自分のチームのドキュメントを見る
1. 上記のチーム一覧から自分のチームをクリック
2. README.md で概要と担当範囲を確認
3. 必要なドキュメントにアクセス

### 🔍 他チームの情報を確認する
1. 連携が必要なチームのフォルダにアクセス
2. README.md で概要を把握
3. 必要に応じて具体的ドキュメントを参照

### 📋 プロジェクト全体を把握する
1. `/teams/docs/` で基本情報を確認
2. 各チームのREADME.md で現状を把握
3. PMチームの進捗管理ドキュメントで全体状況を確認

### 🚨 緊急時の対応
1. `/teams/docs/emergency-messages/` で緊急時対応を確認
2. 担当チームのREADME.md で連絡先を確認
3. PMチーム（中井健登）に第一報

## 技術仕様

### ファイル構造
```
/teams/
├── pm/              # プロジェクト管理
├── content/         # コンテンツ制作
├── marketing/       # マーケティング・セールス
├── design/          # デザイン・ビジュアル
├── live/            # 配信・動画
├── x/               # X/SNS マーケティング
└── docs/            # 共通ドキュメント
```

### 更新・管理ルール
- **更新頻度**: 週1回以上の更新推奨
- **ファイル命名**: 日付-内容.md (例: 20250813-progress-report.md)
- **バージョン管理**: Git での変更履歴管理
- **レビュー**: チーム内レビュー → PMチーム承認

## 連絡先・サポート

### プロジェクト統括
- **PM**: 中井健登
- **Discord**: PMチャンネル
- **緊急時**: 直接連絡

### 技術サポート
- **Next.js アプリ**: `/app/` フォルダ
- **Vercel デプロイ**: `npm run build && npm run start`
- **GitHub Pages**: 静的サイト生成

### ドキュメント管理
- **編集**: Markdown (.md) ファイル
- **画像**: `/assets/` フォルダに格納
- **バックアップ**: Git での自動バックアップ
